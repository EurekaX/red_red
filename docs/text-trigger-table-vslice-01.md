# 首个 Vertical Slice 文本触发表

适用范围：第一章“刘姥姥进大观园”的首个可玩切片：园门、潇湘馆、宴席戏眼。

目标：把热点互动转成可配置的文本触发脚本。玩家可见的文学文本必须来自《红楼梦》原文，不使用仿写旁白。

## 原文来源与版本策略

- 原文来源：维基文库《紅樓夢》第四十回、第四十一回。
- 第四十回：https://zh.wikisource.org/wiki/紅樓夢/第040回
- 第四十一回：https://zh.wikisource.org/wiki/紅樓夢/第041回
- 当前表格保留原文繁体摘句，避免在制作表阶段因自动转简体引入异文。
- 后续如需统一简体，应单独建立“文本校订表”，记录繁简转换和版本选择。
- 玩家可见正文只使用原文摘句；移动提示、解锁条件、UI 状态说明只作为系统配置备注，不进入正文文本框。

## 文本设计原则

- 不写仿古风、不写现代解释型旁白。
- 每个热点优先绑定一段原文摘句，长度以短句或短段为主。
- 若某个交互没有直接对应原文，玩家可见文本应为空，靠音画、UI 高亮、地图状态完成反馈。
- 选择回响不改写原著，只呈现不同原文摘句或原文视角的再排序。
- 重复点击不重复发放收集物，只重复显示同一原文短句或仅播放音画反馈。

## 验收标准

- 玩家可见文学文本全部来自《红楼梦》原文。
- 每条原文摘句都标注来源回目。
- 没有原文依据的互动，不写仿写文本，只做音画或 UI 状态反馈。
- 重复点击不会反复发放同一收集物。
- 后续如引入不同版本，必须在文本校订表中记录异文。
