# 首个 Vertical Slice 文本触发表

适用范围：第一章“刘姥姥进大观园”的首个可玩切片：园门、潇湘馆、宴席戏眼。

目标：把热点互动转成可配置的文本触发脚本，方便后续进入剧情脚本、前端数据结构和本地化文案管理。

## 文本设计原则

- 单次触发文本以 1-3 句为主，避免长篇阅读打断探索节奏。
- 文本分为四类：舞台旁白、梦中看客观察、人物回声、收集题跋。
- 第一次点击给完整反馈，重复点击给短回声或不重复奖励。
- 动线类文本要温柔提示，不使用“任务”“目标”“点击这里”等现代系统口吻。
- 选择回响不改写原著走向，只改变梦中看客读到的意义层。

## 字段说明

| 字段 | 说明 |
| --- | --- |
| Trigger ID | 后续可作为配置表 key |
| 场景 | 所属场景 |
| 热点 | 被点击对象 |
| 触发条件 | 首次、重复、前置解锁等 |
| 文本类型 | 舞台旁白、观察、人物回声、题跋、选择回响 |
| 示例文本 | 第一版可用文案方向 |
| 反馈 | 伴随的音画或 UI 反馈 |
| 产出 | 收集、解锁、状态变化 |
| 重复规则 | 再点时如何处理 |

## 配置实现建议

后续进入前端实现时，可以把文本触发抽象为数据表：

| 字段 | 示例 |
| --- | --- |
| id | GATE_CURTAIN_01 |
| sceneId | gate |
| hotspotId | curtain |
| condition | first_click |
| type | narration |
| text | 帘子一挑，旧梦像被人重新点了灯。 |
| effects | curtain_open, gong_soft |
| unlocks | dream_entered |
| repeatId | GATE_CURTAIN_REPEAT |

## 验收标准

- 每个 P0 热点都有首触发文本。
- 每个场景至少有 1 个收集文本。
- 宴席戏眼至少有 3 个选择回响文本。
- 文本不使用现代任务系统口吻。
- 重复点击不会反复发放同一收集物。
