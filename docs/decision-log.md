# 决策记录

## 2026-06-20

### 目标用户

选择：已经喜欢《红楼梦》的人。

影响：不需要把核心体验做成入门科普，而应更重视审美、细节、象征、人物关系和文本余味。

### 艺术方向

选择：戏台梦境。

影响：美术和演出可以使用帘幕、灯影、曲文、判词、舞台调度、梦境转场等语言。

### 主玩法

选择：游历梦场。

补充：体验类似橙光文字游戏，但要能移步移景，互动感更强，不只限制于文游。

### 第一章切入点

选择：刘姥姥进大观园。

影响：使用外来视角观看大观园繁华，同时保留幽默、礼法、阶层、盛衰暗影等层次。

### 玩家身份

选择：梦中看客。

影响：玩家既能旁观，也能进入刘姥姥视角；既有代入感，又不被单一角色行动范围限制。

### 空间结构

选择：园林地图式。

影响：第一版要先建立可移动、可点击、可回访的地图节点，而不是单线文字分幕。

### 首个 Vertical Slice

选择：先细化三个节点：园门、潇湘馆、宴席戏眼。

影响：用小范围可玩切片验证入梦开场、静态细读、群像高潮三类体验，避免一开始铺完整大观园导致范围失控。

### 首批人物资产策略

选择：第一版优先使用舞台剪影、局部高光和少量关键动作，而不是完整多表情立绘。

影响：可以更快验证戏台梦境气质和互动阅读体验，同时为后续角色精修保留空间。

## 2026-06-21

### 首批热点策略

选择：首个可玩切片先做 13 个 P0 热点，覆盖园门、潇湘馆、宴席戏眼。

影响：用热点验证三种节奏：园门教学、潇湘馆细读、宴席戏眼高潮；每个 P0 热点都必须有视觉反馈、文本反馈和明确功能。

### 文本触发策略

选择：玩家可见的文学文本必须来自《红楼梦》原文，不使用仿写旁白；必要的导航提示只作为系统配置备注，不进入正文文本框。

影响：后续前端可以用配置表驱动原文触发，并保留回目来源；如需繁简转换或版本切换，必须单独建立文本校订表。

### UI 信息架构策略

选择：沉浸场景优先，UI 退到边缘；原文题签、地图、收集册和短选择戏牌构成首版核心 UI。

影响：后续视觉稿和前端实现都应避免现代任务面板感，优先使用题签、戏牌、册页、印记等贴合“戏台梦境”的界面语言。
